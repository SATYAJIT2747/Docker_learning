# 1. Why Docker?

## Problem Before Docker

Imagine you build an ML application.

You send it to your friend.

```

```


Why?

Because every computer has

* Different OS
* Different Python
* Different Libraries
* Different Environment Variables

This problem is called

> **Environment Dependency**

# Solution

Docker packages

* Application
* Python
* Libraries
* Dependencies
* Environment

into one package.

That package is called

# Docker Image

---

Instead of sending code

You send

```

```

Then everyone runs

```

```

and it works exactly the same.

# 26. Most Asked Interview Questions

### Q1. What is Docker?

A platform to build, package, ship, and run applications consistently using containers.

---

### Q2. Difference between Image and Container?

Image is a blueprint.

Container is a running instance of an image.

---

### Q3. Why Docker?

To eliminate "works on my machine" problems by packaging the application and its dependencies into a portable image.

---

### Q4. What is Docker Engine?

The core runtime that builds images, creates and manages containers, networks, and volumes. It receives requests from the Docker CLI (or other clients) through the Docker API.

---

### Q5. What is Docker Hub?

A cloud registry for storing and sharing Docker images.

---

### Q6. What is Repository?

A named collection of Docker image versions (tags). Example: `satyajitpriyadarshi/repo1`.

---

### Q7. What is Tag?

A version label for an image, such as `latest`, `v1.0`, or `python3.12`.

---

### Q8. What is Docker CLI?

The command-line client that sends commands to the Docker Engine.

---

### Q9. What is Docker Daemon?

The background service (`dockerd`) that manages images, containers, networks, and volumes.

---

### Q10. Does the Docker CLI directly create containers?

No. The CLI sends API requests to the Docker Engine, and the Engine performs the actual work.
